# Suika Game — Game Design Document

> 버전 1.0 | 작성일 2026-05-12

---

## 1. 게임 개요

**한 줄 요약:** 과일을 컨테이너 안에 떨어뜨려 같은 종류끼리 합쳐 수박을 만드는 물리 퍼즐 게임

**코어 루프:**

```
과일 선택(미리보기) → 위치 조정(마우스) → 드롭 → 물리 낙하 → 충돌 머지
→ 점수 획득 → 다음 과일 준비 → (반복)
                              ↓
                    과일이 위험선 2초 초과 → 게임 오버
```

**장르:** 캐주얼 물리 퍼즐  
**플랫폼:** PC (Unity 2D, URP)  
**타겟 플레이 시간:** 2~10분/세션

---

## 2. 핵심 메커닉

### 2-1. 떨어뜨리기

- 화면 상단 고정 위치(FruitSpawner)에서 마우스 X 좌표를 따라 과일 미리보기가 이동
- 컨테이너 벽 안쪽 범위(`minX ~ maxX`)로 클램프되어 벽 밖 투척 불가
- 마우스 좌클릭으로 드롭, 드롭 후 `dropCooldown`(기본 0.5초) 동안 다음 과일 준비

### 2-2. 물리 충돌

- 과일은 `Rigidbody2D Dynamic` + `CircleCollider2D` 사용
- 미리보기 상태: Kinematic + 콜라이더 OFF + 반투명(alpha 0.55)
- 드롭 후 `Activate()` 호출 → Dynamic 전환 + 콜라이더 ON

### 2-3. 머지

- 같은 레벨 과일 두 개가 `OnCollisionEnter2D`로 충돌
- `instanceID`가 낮은 쪽이 처리 주도 (동시 이중처리 방지)
- 두 과일의 중간 지점에 다음 레벨 과일 스폰 (`WaitForFixedUpdate` 딜레이)
- 점수: 머지된 두 과일의 `score` 합산
- 수박(레벨 10)끼리 충돌 시 제거만 되고 스폰 없음

### 2-4. 게임 오버

- 컨테이너 상단 위험선(DangerZone, BoxCollider2D Trigger) 위에
- **활성화된(isActive)** 과일이 `gameOverDelay`(기본 2초) 이상 머무르면 게임 오버
- 게임 오버 시: FruitSpawner 비활성, 게임 오버 패널 표시

## 3. 과일 단계

총 **11단계** (레벨 0 → 10), 같은 레벨 2개 합치면 레벨+1 과일 생성.

| 레벨 | 과일 | 반지름(radius) | 점수(score) | 폴백 색상 |
|------|------|---------------|-------------|-----------|
| 0 | 체리 (Cherry) | 0.280 | 1 | #CC1A1A |
| 1 | 딸기 (Strawberry) | 0.392 | 3 | #FF3366 |
| 2 | 포도 (Grape) | 0.504 | 6 | #7B2FBE |
| 3 | 귤 (Tangerine) | 0.616 | 10 | #FF6600 |
| 4 | 감 (Persimmon) | 0.728 | 15 | #FF4500 |
| 5 | 사과 (Apple) | 0.868 | 21 | #CC0000 |
| 6 | 배 (Pear) | 1.036 | 28 | #FFDD00 |
| 7 | 복숭아 (Peach) | 1.218 | 36 | #FFB347 |
| 8 | 파인애플 (Pineapple) | 1.400 | 45 | #FFCC00 |
| 9 | 멜론 (Melon) | 1.610 | 55 | #66BB6A |
| 10 | 수박 (Watermelon) | 1.820 | 66 | #1A801A |

> **크기 비율:** 체리(0.280) → 수박(1.820), 약 6.5배 차이.  
> `radius` 값은 `transform.scale = radius * 2`로 반영됨 (콜라이더는 radius=0.5 고정).  
> `maxRandomSpawnLevel = 4`이므로 랜덤 드롭은 체리~감 5종으로 제한.

## 4. 점수 & 승패 규칙

### 점수

- 머지 발생 시 `scoreA.score + scoreB.score` 즉시 합산
- 연쇄 머지 발생 시 각 머지마다 개별 합산
- 점수 상한 없음
- 게임 오버 시 최종 점수 표시 (재시작 가능)

### 승리 조건

- 없음 (엔드리스). 수박 달성은 보람 목표.

### 패배 조건

- 활성 과일이 위험선 위에 2초 초과 체류

## 5. 화면 & UI 구성

```
┌──────────────────────────────┐
│  점수: 0          다음: 체리  │  ← HUD (Canvas, Screen Space Overlay)
│                              │
│          ○  ← 미리보기        │
│  │                      │    │
│  │    (컨테이너 내부)    │    │
│  │                      │    │
│  └──────────────────────┘    │
│  ─ ─ ─ ─ ─ 위험선 ─ ─ ─ ─ ─  │
└──────────────────────────────┘
         [게임 오버 패널]
         최종 점수: 000
         [다시 시작]
```

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| `scoreText` | TMP_Text | 현재 점수 (형식: `점수: 1,234`) |
| `nextFruitImage` | Image | 다음 드롭 과일 스프라이트 |
| `nextFruitName` | TMP_Text | 다음 드롭 과일 이름 |
| `gameOverPanel` | GameObject | 게임 오버 시 활성화 |
| `finalScoreText` | TMP_Text | `최종 점수\n{score:N0}` |

## 6. 톤 & 분위기

**색감:** 채도 높은 과일 원색 계열. 배경은 크림 화이트 또는 연한 베이지로 과일이 돋보이게.

**아트 방향:**
- 귀여운 2D 플랫 스타일 (그림자 없이 단색 or 단순 그라디언트)
- 스프라이트 없을 때 폴백으로 색상 원 자동 생성 → 프로토타입에서 플레이 가능

**이펙트 (미구현, 향후 추가 고려):**
- 머지 시 작은 파티클 버스트
- 드롭 시 약한 튕김 애니메이션

**BGM / SFX:**
- BGM: 밝고 반복적인 잔잔한 팝 (루프)
- SFX: 드롭 `pop`, 머지 `ding`, 게임 오버 `fail`
